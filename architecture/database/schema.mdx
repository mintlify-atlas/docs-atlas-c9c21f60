## Schema Overview

The Muebles Roble database schema is designed to support a furniture management system with catalogs for colors, wood types, roles, units of measure, and furniture types.

|--------|------|-------------|-------------|
| `id_color` | INTEGER | PRIMARY KEY | Unique identifier |

| `active` | BOOLEAN | NOT NULL, DEFAULT TRUE | Active status |
| `created_at` | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP | Creation timestamp |
| `updated_at` | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | Last update timestamp |
| `deleted_at` | TIMESTAMP | NULL | Soft delete timestamp |

## Common Schema Patterns

### Standard Field Structure

All tables follow a consistent field pattern:

<Tabs>
  <Tab title="Primary Key">
    ```sql
    id_{table_name} INTEGER PRIMARY KEY AUTO_INCREMENT
    ```
    
    Each table uses an integer primary key with a descriptive name.
  </Tab>
  
  <Tab title="Audit Timestamps">
    ```sql
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP 
               ON UPDATE CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP NULL
    ```
    
    Automatic timestamp tracking for all record changes.
  </Tab>
  
  <Tab title="User Tracking">
    ```sql
    created_by VARCHAR(100) NULL,
    updated_by VARCHAR(100) NULL,
    deleted_by VARCHAR(100) NULL
    ```
    
    Tracks which user performed each action.
  </Tab>
  
  <Tab title="Soft Delete">
    ```sql
    active BOOLEAN NOT NULL DEFAULT TRUE,
    deleted_at TIMESTAMP NULL
    ```
    
    Enables soft deletion of records.
  </Tab>
</Tabs>

### Naming Conventions

<AccordionGroup>
  <Accordion title="Table Names">
    - Use **plural** form: `colors`, `roles`, `wood_types`
    - Use **snake_case**: `unit_of_measures`, not `UnitOfMeasures`
    - Be descriptive and specific
  </Accordion>
  
  <Accordion title="Column Names">
    - Use **snake_case**: `id_color`, `created_at`
    - Primary keys: `id_{table_name}` (e.g., `id_color`)
    - Foreign keys: `id_{referenced_table}` (e.g., `id_color` in furniture table)
    - Booleans: Use affirmative names (`active`, not `inactive`)
  </Accordion>
  
  <Accordion title="Constraint Names">
    - Primary keys: `pk_{table_name}`
    - Foreign keys: `fk_{table}_{referenced_table}`
    - Unique constraints: `uq_{table}_{column}`
    - Indexes: `idx_{table}_{column}`
  </Accordion>
</AccordionGroup>

## Schema Evolution

### Adding New Tables

When adding a new table, follow this template:

```sql
CREATE TABLE new_table (
    id_new_table INTEGER PRIMARY KEY AUTO_INCREMENT,
    
    -- Business fields
    name VARCHAR(100) NOT NULL,
    description VARCHAR(255) NULL,
    
    -- Status
    active BOOLEAN NOT NULL DEFAULT TRUE,
    
    -- Audit timestamps
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP 
               ON UPDATE CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP NULL,
    
    -- User tracking
    created_by VARCHAR(100) NULL,
    updated_by VARCHAR(100) NULL,
    deleted_by VARCHAR(100) NULL,
    
    -- Constraints
    UNIQUE KEY uq_new_table_name (name)
);
```

### Adding Relationships

When adding foreign keys:

```sql
-- Add foreign key column
ALTER TABLE furniture 
ADD COLUMN id_color INTEGER NULL;

-- Add constraint
ALTER TABLE furniture
ADD CONSTRAINT fk_furniture_color 
FOREIGN KEY (id_color) REFERENCES colors(id_color)
ON DELETE SET NULL
ON UPDATE CASCADE;

-- Add index for performance
CREATE INDEX idx_furniture_color ON furniture(id_color);
```

## Database Configuration

### Connection Settings

Database configuration is managed through environment variables:

```python config.py
import os

class Config:
    DB_USER = os.getenv('DB_USER')
    DB_PASSWORD = os.getenv('DB_PASSWORD')
    DB_HOST = os.getenv('DB_HOST', 'localhost')
    DB_PORT = os.getenv('DB_PORT', '3306')
    DB_NAME = os.getenv('DB_NAME')
    
    SQLALCHEMY_DATABASE_URI = (
        f"mysql+pymysql://{DB_USER}:{DB_PASSWORD}@"
        f"{DB_HOST}:{DB_PORT}/{DB_NAME}"
    )
    SQLALCHEMY_TRACK_MODIFICATIONS = False
```

## Query Performance

### Recommended Indexes

All tables have these indexes by default:

```sql
-- Primary key (automatic)
PRIMARY KEY (id_{table_name})

-- Unique constraint on name
UNIQUE KEY uq_{table}_name (name)

-- Performance indexes
INDEX idx_{table}_active (active)
INDEX idx_{table}_created_at (created_at)
```

### Query Optimization

<Tabs>
  <Tab title="Filter Active Records">
    Always filter by `active` status:
    
    ```python
    # Good - uses index
    colors = Color.query.filter_by(active=True).all()
    
    # Avoid - scans all records
    colors = [c for c in Color.query.all() if c.active]
    ```
  </Tab>
  
  <Tab title="Case-Insensitive Search">
    Use database functions for case-insensitive queries:
    
    ```python
    from sqlalchemy import func
    
    # Case-insensitive search
    color = Color.query.filter(
        func.lower(Color.name) == name.lower()
    ).first()
    ```
  </Tab>
  
  <Tab title="Count Efficiently">
    Use COUNT queries instead of loading all records:
    
    ```python
    # Good - database COUNT
    count = Color.query.filter_by(active=True).count()
    
    # Avoid - loads all records
    count = len(Color.query.filter_by(active=True).all())
    ```
  </Tab>
</Tabs>
